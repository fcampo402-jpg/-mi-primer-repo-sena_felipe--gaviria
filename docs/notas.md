   - ¿Para qué crees que sirve un archivo .gitignore?

Un archivo .gitignore sirve para indicarle a Git qué archivos o carpetas no debe guardar ni subir al repositorio.
Por ejemplo, se usa para evitar subir archivos temporales, configuraciones personales o archivos grandes que no son necesarios para el proyecto. Así el repositorio se mantiene limpio y solo contiene lo importante para el código.

   - ¿Por qué es importante escribir buenos mensajes en los commits?

Es importante escribir buenos mensajes en los commits porque ayudan a entender claramente qué cambios se hicieron y por qué. Esto facilita el trabajo en equipo, la revisión del código y encontrar errores en el futuro. Un mensaje claro permite que cualquier persona (incluso tú mismo después de un tiempo) sepa qué se modificó sin tener que revisar todo el código.
Ejemplo de cómo agregarlo:
git commit -m "Corrige error en el formulario de registro de usuarios"
